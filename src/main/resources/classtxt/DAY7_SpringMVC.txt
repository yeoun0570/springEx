1. 스프링 Web MVC 기초
    a. 스프링 프레임워크
        i. 사용 이유 : 개발의 생산성과 안정성
    b. 스프링 웹 MVC 특징
        i. 스프링 웸 MVC는 Front_Controller 패턴(facade 패턴)을 이용해서 사전/사후 처리를 가능하도록 설계되어있다.
            1. facade 패턴 : 모든 흐름이 하나의 객체를 통해서 진행된다. 웹구조에서는 Front-Controller 패턴이라고 불려진다.
            2. Front-Controller를 이용하면 모든 요청이 '반드시' 하나의 객체를 지나서 처리되기 때문에 모든 공통적인 처리를 프론트 컨트롤러에서 처리할 수 있다.
        ii. 어노테이션을 적극적으로 활용하여 최소한의 코드로 많은 처리를 가능하도록 설계되어 있다.
        iii. HttpServletRequest / HttpServletResponse 를 이용하지 않아도 될만큼 추상화된 방식으로 개발이 가능하다.
        iv. 사전/사후
    c. 기존 MVC와의 차이점
        i. 상속이나 인터페이스를 구현하는 방식을 사용하지 않고 어노테이션만으로 처리 가능
        ii. 오버라이드 없이 필요한 메서드를 정의
        iii. 메서드의 파라미터를 기본 자료형이나 객체 자료형을
